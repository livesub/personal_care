# Dry Run: 로그인 성공 후 뒤로가기 시나리오

## 검증 목표
로그인 성공 후 **뒤로가기**를 눌렀을 때 **로그인 화면으로 돌아가지 않고 메인(홈)에 남는지** 코드 시뮬레이션으로 확인.

---

## 1. 현재 코드 요약

| 구분 | 코드 위치 | 동작 |
|------|-----------|------|
| 로그인 성공 시 | `new_login_screen.dart` | `auth.setSession(response)` → `notifyAuthChange()` → `context.go('/home')` |
| 라우터 가드 | `router.dart` `redirect` | `location == '/login'` && `isLoggedIn` → `return '/home'` |
| go() 사용처 | 전부 | 스플래시·역할선택·로그인·홈 모두 `context.go(...)` 사용 (push 없음) |

---

## 2. 가상 시나리오 (Dry Run)

### 시나리오 A: 로그인 성공 직후 스택 상태

1. 사용자: 역할선택 → 보호사 탭 → `context.go('/login?role=helper')`
2. 로그인 성공: `context.go('/home')` 실행
3. **go_router 동작**: `go()`는 **현재 위치를 대체**하므로, 스택은 `[/home]` 한 칸만 갖거나, 히스토리상 “이전 경로”가 `/login`으로 남을 수 있음 (플랫폼/버전 의존).

**가정**: 뒤로가기 시 GoRouter가 “이전 경로”로 이동한다고 가정 → **location 이 `/login`(또는 `/`)으로 바뀜**.

---

### 시나리오 B: 홈에서 뒤로가기 → 이전 경로가 `/login`인 경우

| 단계 | 이벤트 | 라우터 상태 | redirect 호출 |
|------|--------|-------------|----------------|
| 1 | 사용자가 홈 화면에서 뒤로가기 | 이동 시도: `/login` | 호출됨 |
| 2 | redirect 진입 | `location == '/login'`, `isLoggedIn == true` | - |
| 3 | 가드 조건 | `if (location == '/login' \|\| location == '/role-selection') { if (isLoggedIn) return '/home'; }` | **return '/home'** |
| 4 | 결과 | 라우터가 `/home`으로 다시 리다이렉트 | **로그인 화면 미노출, 메인 유지** ✅ |

**결론**: 뒤로가기로 `/login`으로 “들어가려” 해도 redirect가 즉시 `/home`을 반환하므로 **로그인 화면으로 돌아가지 않고 메인에 남는다.**

---

### 시나리오 C: 홈에서 뒤로가기 → 이전 경로가 `/`(스플래시)인 경우

앱 기동 시 `initialLocation: '/'`, 스플래시에서 `context.go('/role-selection')` 등으로 이동하므로, 일부 환경에서는 **뒤로가기 시 location 이 `/`로 복귀**할 수 있음.

| 단계 | 이벤트 | 라우터 상태 | redirect 동작 |
|------|--------|-------------|----------------|
| 1 | 홈에서 뒤로가기 | 이동 시도: `/` | redirect 호출 |
| 2 | redirect | `location == '/'`, `isLoggedIn == true` | 현재 코드: `'/'`에 대한 분기 **없음** → `return null` |
| 3 | 결과 | **스플래시 화면(`/`)이 그대로 표시됨** | ❌ 구멍 |

**결론**: 로그인된 사용자가 홈에서 뒤로가기를 눌렀을 때 **스플래시로 돌아가는 구멍**이 있음.

---

## 3. 로직 구멍 요약

| 시나리오 | 뒤로가기 목적지 | 현재 동작 | 로그인 화면 노출 여부 |
|----------|------------------|-----------|------------------------|
| B | `/login` | redirect → `/home` | ✅ 노출 안 됨 (의도 충족) |
| C | `/` | redirect 없음 → 스플래시 표시 | ⚠️ 스플래시 노출 (의도에서 벗어남) |

- **“로그인 화면으로 안 돌아간다”** 는 요구사항은 **시나리오 B**에서 만족.
- **“메인에 남는다”** 륄 넓게 보면, **시나리오 C**에서 스플래시로 나가는 것은 구멍.

---

## 4. 수정 제안

**router.dart `redirect`** 에 다음 분기 추가:

- `location == '/'` 이고 `isLoggedIn == true` 이면 **`return '/home'`**
- 로그인된 사용자가 뒤로가기로 `/`에 들어오더라도 곧바로 메인으로 돌려보냄.

적용 후 기대:

- 뒤로가기 → `/login` → redirect → `/home` ✅
- 뒤로가기 → `/` → redirect → `/home` ✅ (구멍 제거)

---

## 5. 반영된 수정 (router.dart)

- `redirect` 조건에 **`location == '/'`** 를 추가하여, 로그인된 상태에서 `/`(스플래시)로 들어오면 즉시 `'/home'`으로 리다이렉트하도록 수정함.
